Doctor Appointment System

Book appointments. Manage schedules. Simplify healthcare workflow.

Doctor Appointment System is a full-stack web application built with React (frontend), Node.js + Express (backend), and MongoDB (database). It supports patient booking, doctor availability, admin management, authentication, and appointment lifecycle tracking.

Project Overview

This system helps clinics/hospitals manage appointments and reduce manual scheduling. Patients can register, browse doctors, pick a slot, and track bookings.

Doctors can manage profiles, set availability, view upcoming appointments, and update status (approved, completed, canceled).

Admin can manage doctors, patients, specialties, and appointment records from a central dashboard.

3 Roles: Patient, Doctor, Admin
2 Apps: Frontend + Backend API
1 Database: MongoDB
1 Central Theme: Deep Sky Blue + White

Key Features

React Frontend

Modern UI for booking, dashboards, forms, and role-based screens.

Node.js + Express API

REST API for auth, doctors, appointments, availability, and admin actions.

MongoDB Database

Mongoose models for Users, Doctors, Schedules, Appointments, and Reviews.

Authentication & Roles

JWT-based login and protected routes for Patient / Doctor / Admin.

Appointment Booking

Choose doctor, date, time slot, reason, and confirm booking.

Status & Notifications

Track booking status (Pending / Approved / Completed / Cancelled).

Doctor Availability

Create weekly schedules, slot limits, and holidays/unavailable days.

Admin Dashboard

Manage doctors, specialties, users, and appointment history from one panel.

Validation & Security

Input validation, API rate limiting (optional), and secure env configs.

Installation & Run

Prerequisites

  • Node.js (LTS recommended)
  • MongoDB (local or Atlas)
  • npm or yarn

Backend Setup (Node + Express) 

cd server
npm install

# create .env
# see env section below

npm run dev

Backend will start on something like http://localhost:5000

Frontend Setup (React) 

cd client
npm install

# (optional) create .env
# VITE_API_URL=http://localhost:5000 or REACT_APP_API_URL=...

npm start

Frontend will start on http://localhost:3000 (React) or 5173 (Vite).

Environment Variables (.env) 

# server/.env
PORT=5000
MONGO_URI=mongodb+srv://USER:PASSWORD@cluster.mongodb.net/doctor-appointments
JWT_SECRET=your_super_secret_key

# optional
CORS_ORIGIN=http://localhost:3000

API Overview (Example)

Your endpoints may vary, but this is a clean structure to document:

Auth

  • POST /api/auth/register
  • POST /api/auth/login
  • GET /api/auth/me

Doctors

  • GET /api/doctors
  • GET /api/doctors/:id
  • POST /api/doctors (admin)

Appointments

  • POST /api/appointments
  • GET /api/appointments/my
  • PATCH /api/appointments/:id/status
